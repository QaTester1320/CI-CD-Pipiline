These are the Execution Engines. When a developer pushes code, these tools wake up, rent a computer in the cloud, and follow the instructions in your YAML file.

‚Ä¢ Jenkins: Self-hosted, very customizable.
‚Ä¢ GitHub Actions: Built into GitHub, very easy to use.

üìÑ The YAML File (.yml)

This is the Contract. It tells the Engine exactly what to do. Without this file, the pipeline has no brain.

jobs:
  build:
    runs-on: ubuntu-latest
    steps:
      - name: Compile Code
        run: npm run build

üöÄ Manual vs. CI/CD

Manual: A person builds the app, runs tests by hand, and uploads files via FTP. Slow and full of human error.

CI/CD: A robot does it in seconds. If a test fails, the robot stops the release automatically, protecting the customers.
